## 📋 Test Plan

### Why We Need It

**Business:**
- Defines what will and will not be tested — sets stakeholder expectations
- Provides a clear timeline and effort estimate for release planning
- Reduces financial risk by identifying high-risk areas before development
- Gives management confidence that quality is controlled and measurable

**Development Team:**
- Aligns QA and development on scope before a single line of code is written
- Identifies dependencies and blockers early — no surprises during testing
- Defines entry and exit criteria so everyone agrees when testing is done
- Prioritizes testing effort on highest-risk features first

---

### Aim
- Define scope, approach, and effort before testing begins
- Identify and prioritize risk across all features in the release
- Set clear entry and exit criteria agreed upon by all stakeholders
- Serve as a single source of truth for the entire testing cycle

---

### Deliverables
- Scope definition (in scope / out of scope)
- Risk assessment table
- Test approach and strategy
- Entry and exit criteria
- Effort estimation
- Roles and responsibilities
- Test environment requirements
